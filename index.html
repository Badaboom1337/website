Git — система контролю версій

Що таке система контролю версій?

Системи контролю версій (VCS) — це програмні інструменти, які дозволяють відслідковувати зміни у файлах, керувати версіями коду та спільною роботою. Найпопулярнішою є Git.

Переваги Git

  • Розподілена система — кожен розробник має повну копію репозиторію.
  • Висока швидкість роботи.
  • Підтримка гілок (branching) для паралельної розробки.
  • Надійність та безпека даних.
  • Широке використання у світі (GitHub, GitLab, Bitbucket).

10 базових команд Git

  1. git init — створення нового репозиторію.
  2. git clone [url] — клонування існуючого репозиторію.
  3. git status — перевірка стану файлів.
  4. git add [файл] — додавання файлу у staging.
  5. git commit -m "повідомлення" — збереження змін.
  6. git log — історія комітів.
  7. git branch — перегляд імен гілок.
  8. git checkout [гілка] — перемикання між гілками.
  9. git merge [гілка] — злиття гілок.
  10. git push / git pull — відправка чи отримання змін з віддаленого репозиторію.
